Submissions (Process 2.0)

  • Generates unique Tracking ID
  • Stores subject, kind, office, and attachments
  • Initial Status: PENDING

Governance (Process 3.0)

  • Admin provides public remarks for feedback
  • Transitions: APPROVED, DECLINED, RETURNED
  • Triggers stakeholder notifications

Compliance (Process 4.0)

  • Assignees only see APPROVED records
  • Mark as Completed transitions to final status
  • Alerts both Submitter and Admin of success
