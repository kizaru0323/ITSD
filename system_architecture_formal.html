1. Logging In

Users enter their username and password. The system checks their account and takes them to the right dashboard (Admin, Staff, or Worker).

2. Sending a Request

Staff members fill out the task form and upload files. The system gives it a unique ID and saves it.

3. Admin Checking

The Admin looks at the new requests and decides to Approve them for work, Return them for fixes, or Decline them.

4. Doing the Work

Once a task is approved, it goes to the Worker. The Worker does the job and clicks a button to say they are finished.

5. Sending Alerts

The system sends messages automatically whenever a task is approved, returned, or finished so everyone knows the status.

6. Saving for History

After a task is finished, it is saved forever in the history so the Admin can see reports later.

Simple Flow Summary

  • Saving Data: When Staff submits, the info goes into the Tasks Database.
  • Changing Status: The Admin's check is what moves the task from 'New' to 'Approved'.
  • Automated Alerts: The Alert Engine sends messages automatically so no one stays guessing.
  • Stats & History: The Report Hub keeps track of all changes and shows them to the Admin in charts.
